# Security Policy

## 対象

- Webダッシュボード
- ファイルアップロード
- Raspberry Piのネットワーク接続
- 認証情報・APIキー
- 訓練映像・解析データ
- 学習済みモデル

## 報告方法

脆弱性や個人情報漏えいにつながる問題は、公開Issueへ詳細を書かないでください。  
GitHubのPrivate Vulnerability Reportingが有効な場合は、そちらを利用してください。

## 基本ルール

- APIキー、パスワード、秘密鍵をコミットしない
- `.env`をGit管理しない
- Raspberry Piの初期パスワードを変更する
- 不要なポートを公開しない
- 映像の保存先とアクセス権を限定する
- 公開前にメタデータを確認する
- 外部依存パッケージを定期的に更新する
