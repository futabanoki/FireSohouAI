# Contributing to FireSohouAI

ご参加ありがとうございます。

FireSohouAIでは、コードだけでなく、現場知識、検証結果、ドキュメント、映像設計、プライバシー改善なども重要な貢献として扱います。

## 参加できる内容

- 消防操法の現場課題や改善案
- 教範・実施要領との整合性確認
- Sora等の映像プロンプト
- Raspberry Pi撮影プログラム
- MediaPipe姿勢解析
- YOLO器具・イベント認識
- 評価・可視化・ダッシュボード
- テスト、文書修正、翻訳
- プライバシー・安全面の改善

## Issueを作る前に

既存Issueを検索し、同じ内容がないか確認してください。

現場で発生した課題は、可能な範囲で次を記載してください。

- 発生日または訓練日
- 対象役割
- 対象動作
- 期待する状態
- 実際に起きた状態
- 再現条件
- 映像やログの有無
- 個人情報を除去したか

## ブランチ名

```text
feature/<機能名>
fix/<修正内容>
docs/<文書名>
experiment/<実験名>
```

例:

```text
feature/pose-angle-export
fix/video-timestamp
docs/scene03
experiment/yolo-nozzle-detection
```

## コミットメッセージ

短く、変更理由が分かる表現にします。

```text
docs: add project vision
feat: add pose angle calculator
fix: correct timestamp conversion
test: add capture frame-rate test
```

## Pull Request

Pull Requestには次を記載してください。

- 変更内容
- 変更理由
- 確認方法
- 影響範囲
- 関連Issue
- 映像・個人情報の有無

## コード方針

- Pythonは読みやすさを優先する
- 型ヒントを可能な範囲で付ける
- 外部入力には検証と例外処理を入れる
- ハードウェア依存値を設定ファイルへ分離する
- 解析結果には単位を明記する
- 乱数を使う処理は再現条件を記録する
- 学習モデルの出典・ライセンスを記録する

## 現場データ

訓練映像や測定データをコミットする前に、必ず次を確認してください。

- 撮影対象者の同意
- 所属組織の許可
- 顔・氏名・所属・位置情報の扱い
- 音声に含まれる個人情報
- Git履歴に残して問題がないか
- 公開リポジトリに置く必要が本当にあるか

原則として、元の訓練映像や個人情報を含むデータは公開リポジトリへ直接置かないでください。

## レビューの考え方

消防操法としての正確性と、ソフトウェアとしての正確性は別々に確認します。

- 現場レビュー: 動作、安全、教範との整合性
- 技術レビュー: 実装、再現性、性能、保守性
- データレビュー: 同意、匿名化、ライセンス
