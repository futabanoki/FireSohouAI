\# FireSohouAI Version 0.1  

\# 動画ファイル命名規則



Version: 0.1.0  

Status: Draft  

Target Issue: Issue #3 - Define video file naming rules  

Last Updated: 2026-07-19



\---



\## 1. 目的



本書は、FireSohouAIで撮影・解析する動画ファイルの命名規則を定義する。



命名規則を統一することで、次を実現する。



\- 撮影日や対象役割をファイル名から判別できる

\- 同一条件の撮影データを整理しやすくする

\- MediaPipeやYOLOの解析処理で自動分類しやすくする

\- 訓練映像同士を比較しやすくする

\- 重複ファイルや上書きを防止する

\- GitHubやローカル環境での管理を安定させる

\- 将来のデータセット作成に利用できる



\---



\## 2. 基本方針



動画ファイル名は、次の条件を満たすものとする。



1\. 半角英数字とアンダースコアのみを使用する

2\. 日本語、空白、記号を使用しない

3\. 撮影日を先頭に記録する

4\. 対象役割を記録する

5\. 撮影番号を記録する

6\. 同じ名前のファイルを作らない

7\. ファイル名は小文字に統一する

8\. 拡張子は `.mp4` を基本とする



\---



\## 3. 標準ファイル名



Version 0.1の標準形式は次のとおりとする。



```text

yyyymmdd\_role\_session\_take\_camera.mp4

```



\### 例



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



このファイル名は、次の意味を持つ。



| 項目 | 値 | 意味 |

|---|---|---|

| 撮影日 | 20260719 | 2026年7月19日 |

| 役割 | no1 | 一番員 |

| セッション | s01 | 当日の1回目の訓練 |

| テイク | t01 | そのセッションの1回目の撮影 |

| カメラ | cam01 | 1台目のカメラ |

| 拡張子 | mp4 | MP4動画 |



\---



\## 4. ファイル名の構成



```text

\[date]\_\[role]\_\[session]\_\[take]\_\[camera].\[extension]

```



各項目はアンダースコア `\_` で区切る。



\---



\## 5. 撮影日



\### 5.1 形式



```text

yyyymmdd

```



\### 5.2 例



```text

20260719

```



\### 5.3 規則



\- 西暦4桁を使用する

\- 月と日は2桁で記録する

\- 区切り記号は使用しない

\- 撮影した現地の日付を使用する



\### 正しい例



```text

20260719

20261005

20270103

```



\### 誤った例



```text

2026-07-19

2026\_07\_19

260719

2026719

```



\---



\## 6. 役割コード



役割は次のコードを使用する。



| 役割 | コード |

|---|---|

| 指揮者 | commander |

| 一番員 | no1 |

| 二番員 | no2 |

| 三番員 | no3 |

| 4名全体 | team |

| 器具のみ | equipment |

| テスト撮影 | test |



\### 例



```text

20260719\_commander\_s01\_t01\_cam01.mp4

20260719\_no1\_s01\_t01\_cam01.mp4

20260719\_no2\_s01\_t01\_cam01.mp4

20260719\_no3\_s01\_t01\_cam01.mp4

```



Version 0.1では、主解析対象が1名の場合、その対象役割を記録する。



4名全員を広角で撮影した場合は `team` を使用する。



\---



\## 7. セッション番号



\### 7.1 形式



```text

sNN

```



`NN` は2桁の連番とする。



\### 7.2 例



```text

s01

s02

s03

```



\### 7.3 定義



セッションとは、1回のまとまった訓練または撮影単位を表す。



次の場合は、新しいセッション番号を使用する。



\- 訓練を一度終了して再開した

\- カメラ位置を大きく変更した

\- 対象隊員を変更した

\- 撮影条件を大きく変更した

\- 午前と午後で分けて管理したい

\- 別の訓練メニューへ移行した



\### 例



```text

s01

```



午前の訓練



```text

s02

```



午後の訓練



\---



\## 8. テイク番号



\### 8.1 形式



```text

tNN

```



`NN` は2桁の連番とする。



\### 8.2 例



```text

t01

t02

t03

```



\### 8.3 定義



テイクとは、1回の録画開始から録画停止までを表す。



同じセッション内で複数回撮影した場合は、テイク番号を増やす。



\### 例



```text

20260719\_no1\_s01\_t01\_cam01.mp4

20260719\_no1\_s01\_t02\_cam01.mp4

20260719\_no1\_s01\_t03\_cam01.mp4

```



録画に失敗した場合でも、番号を再利用しないことを推奨する。



\---



\## 9. カメラ番号



\### 9.1 形式



```text

camNN

```



`NN` は2桁の連番とする。



\### 9.2 例



```text

cam01

cam02

cam03

```



\### 9.3 定義



| コード | 想定用途 |

|---|---|

| cam01 | Version 0.1の主カメラ |

| cam02 | 側面または補助カメラ |

| cam03 | 正面または予備カメラ |



Version 0.1でカメラが1台の場合も、将来の拡張を考慮して `cam01` を記録する。



\---



\## 10. 拡張子



標準拡張子は次のとおりとする。



```text

.mp4

```



推奨映像形式:



```text

MP4 / H.264

```



カメラが `.mov` などで記録した場合は、元ファイルを保存したうえで、解析用にMP4へ変換する。



\### 元動画



```text

20260719\_no1\_s01\_t01\_cam01\_original.mov

```



\### 変換後動画



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



ただし、通常の標準運用では、ファイル名に `original` を追加せず、保存フォルダで元動画と解析用動画を分離することを推奨する。



\---



\## 11. 標準例



\### 11.1 指揮者



```text

20260719\_commander\_s01\_t01\_cam01.mp4

```



\### 11.2 一番員



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



\### 11.3 二番員



```text

20260719\_no2\_s01\_t01\_cam01.mp4

```



\### 11.4 三番員



```text

20260719\_no3\_s01\_t01\_cam01.mp4

```



\### 11.5 4名全体



```text

20260719\_team\_s01\_t01\_cam01.mp4

```



\### 11.6 カメラ位置比較



```text

20260719\_no1\_s02\_t01\_cam01.mp4

20260719\_no1\_s02\_t01\_cam02.mp4

20260719\_no1\_s02\_t01\_cam03.mp4

```



同じ動作を複数カメラで同時撮影した場合、撮影日・役割・セッション・テイクを同じ値にし、カメラ番号だけを変更する。



\---



\## 12. テスト撮影の命名



本番訓練ではない撮影は、役割コードに `test` を使用する。



\### 例



```text

20260719\_test\_s01\_t01\_cam01.mp4

```



特定役割のテストであることを明確にしたい場合は、標準ファイル名を維持し、メタデータ側で `recording\_type=test` と記録する。



Version 0.1では、ファイル名を複雑にしすぎないため、原則として次のいずれかを使用する。



```text

20260719\_test\_s01\_t01\_cam01.mp4

```



または



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



後者の場合は、撮影記録にテストであることを明記する。



\---



\## 13. 解析後ファイルの命名



解析結果は、元動画の基本名を引き継ぐ。



元動画:



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



拡張子を除いた部分を、ベース名と呼ぶ。



```text

20260719\_no1\_s01\_t01\_cam01

```



\### 13.1 MediaPipe解析動画



```text

20260719\_no1\_s01\_t01\_cam01\_pose.mp4

```



\### 13.2 YOLO解析動画



```text

20260719\_no1\_s01\_t01\_cam01\_yolo.mp4

```



\### 13.3 統合解析動画



```text

20260719\_no1\_s01\_t01\_cam01\_combined.mp4

```



\### 13.4 姿勢角度CSV



```text

20260719\_no1\_s01\_t01\_cam01\_pose\_angles.csv

```



\### 13.5 YOLO検出結果CSV



```text

20260719\_no1\_s01\_t01\_cam01\_detections.csv

```



\### 13.6 解析サマリー



```text

20260719\_no1\_s01\_t01\_cam01\_summary.csv

```



\### 13.7 Markdownレポート



```text

20260719\_no1\_s01\_t01\_cam01\_report.md

```

\## 15. メタデータファイル



動画ファイルと同じベース名を使用し、JSON形式で撮影条件を保存する。



\### 動画



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



\### メタデータ



```text

20260719\_no1\_s01\_t01\_cam01.json

```



\### JSON例



```json

{

&#x20; "file\_name": "20260719\_no1\_s01\_t01\_cam01.mp4",

&#x20; "recording\_date": "2026-07-19",

&#x20; "role": "no1",

&#x20; "session": 1,

&#x20; "take": 1,

&#x20; "camera\_id": "cam01",

&#x20; "recording\_type": "practice",

&#x20; "resolution": "1920x1080",

&#x20; "fps": 30,

&#x20; "codec": "H.264",

&#x20; "camera\_height\_m": 1.4,

&#x20; "camera\_distance\_m": 20.0,

&#x20; "camera\_angle\_deg": 45,

&#x20; "weather": "sunny",

&#x20; "lighting": "front\_light",

&#x20; "notes": ""

}

```

\## 16. 個人識別情報



動画ファイル名には、次の情報を含めない。



\- 氏名

\- ニックネーム

\- 消防団員番号

\- 所属分団名

\- 電話番号

\- メールアドレス

\- その他、個人を直接特定できる情報



個人を区別する必要がある場合は、匿名化した参加者IDをメタデータに記録する。



\### 例



```json

{

&#x20; "participant\_id": "p001"

}

```



ファイル名へ参加者IDを含める必要が生じた場合は、将来の仕様変更として検討する。



Version 0.1の標準ファイル名には含めない。



\---



\## 17. 禁止する文字



ファイル名には次を使用しない。



```text

空白

日本語

全角文字

\-

/

\\

:

\*

?

"

<

>

|

(

)

\[

]

```



Version 0.1では、使用可能な文字を次に限定する。



```text

a-z

0-9

\_

.

```



拡張子のドット以外では、原則としてドットを使用しない。



\---



\## 18. 大文字・小文字



ファイル名はすべて小文字とする。



\### 正しい例



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



\### 誤った例



```text

20260719\_No1\_S01\_T01\_CAM01.MP4

20260719\_NO1\_s01\_t01\_cam01.mp4

```



Windowsでは大文字・小文字を区別しない場合があるが、Linux環境やRaspberry Piでは区別される。



環境間の問題を防ぐため、小文字へ統一する。



\---



\## 19. 連番の桁数



セッション、テイク、カメラは2桁とする。



\### 正しい例



```text

s01

t01

cam01

```



\### 誤った例



```text

s1

t1

cam1

```



2桁に統一することで、ファイル一覧を正しい順番で並べやすくなる。



100件以上必要になった場合は、Version 0.2以降で3桁化を検討する。



\---



\## 20. 重複防止



同じファイル名が存在する場合、既存ファイルを上書きしない。



次のいずれかを確認し、適切な項目を変更する。



\- セッション番号

\- テイク番号

\- カメラ番号

\- 撮影日



通常は、テイク番号を増やす。



\### 例



既存:



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



新規:



```text

20260719\_no1\_s01\_t02\_cam01.mp4

```



`copy`、`new`、`final`、`final2`などの曖昧な文字列は使用しない。



\### 使用しない例



```text

20260719\_no1\_s01\_t01\_cam01\_copy.mp4

20260719\_no1\_final.mp4

20260719\_no1\_final2.mp4

```



\---



\## 21. 撮影途中で失敗した場合



録画失敗、カメラ転倒、画面外、音声不良などがあった場合も、使用したテイク番号は原則として再利用しない。



\### 例



```text

t01

```



録画失敗



```text

t02

```



再撮影



失敗理由はメタデータへ記録する。



```json

{

&#x20; "status": "failed",

&#x20; "notes": "Subject moved outside the frame."

}

```



不要な動画を削除する場合も、撮影記録上は欠番を許容する。



\---



\## 22. ファイル名変更



撮影後にファイル名を変更する場合は、動画と関連ファイルをすべて同じベース名へ変更する。



\### 変更前



```text

video001.mp4

```



\### 変更後



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



関連ファイル:



```text

20260719\_no1\_s01\_t01\_cam01.json

20260719\_no1\_s01\_t01\_cam01\_pose\_angles.csv

20260719\_no1\_s01\_t01\_cam01\_report.md

```



動画だけを変更し、メタデータや解析結果の名前を変更しない状態は避ける。



\---



\## 23. Raspberry Piでの命名



Raspberry Piの撮影スクリプトは、標準命名規則に従ってファイル名を自動生成することを目標とする。



\### コマンド例



```text

python capture\_video.py --role no1 --session 1 --take 1 --camera cam01

```



生成されるファイル:



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



撮影日はRaspberry Piのシステム日付から取得する。



そのため、撮影前に日付と時刻が正しいことを確認する。



\---



\## 24. 命名規則の正規表現



標準動画ファイル名は、次の正規表現に適合するものとする。



```regex

^\\d{8}\_(commander|no1|no2|no3|team|equipment|test)\_s\\d{2}\_t\\d{2}\_cam\\d{2}\\.mp4$

```



\### 適合例



```text

20260719\_no1\_s01\_t01\_cam01.mp4

20260719\_commander\_s02\_t03\_cam02.mp4

```



\### 非適合例



```text

2026-07-19\_no1\_s01\_t01\_cam01.mp4

20260719\_no4\_s01\_t01\_cam01.mp4

20260719\_no1\_s1\_t1\_cam1.mp4

```

\## 25. Pythonでの利用例



ファイル名をアンダースコアで分割することで、主要情報を取得できる。



```python

from pathlib import Path



video\_path = Path("20260719\_no1\_s01\_t01\_cam01.mp4")

parts = video\_path.stem.split("\_")



recording\_date = parts\[0]

role = parts\[1]

session = parts\[2]

take = parts\[3]

camera = parts\[4]



print(recording\_date)

print(role)

print(session)

print(take)

print(camera)

```



出力:



```text

20260719

no1

s01

t01

cam01

```



実際のプログラムでは、正規表現による形式確認とエラー処理を追加する。

\## 26. 命名チェックリスト



撮影後に次を確認する。



\- \[ ] 撮影日が8桁である

\- \[ ] 役割コードが正しい

\- \[ ] セッション番号が2桁である

\- \[ ] テイク番号が2桁である

\- \[ ] カメラ番号が2桁である

\- \[ ] すべて小文字である

\- \[ ] 空白が含まれていない

\- \[ ] 日本語が含まれていない

\- \[ ] 拡張子が `.mp4` である

\- \[ ] 既存ファイルと重複していない

\- \[ ] 個人名が含まれていない

\- \[ ] メタデータファイルとベース名が一致している

\## 27. Version 0.1の標準規則



FireSohouAI Version 0.1の標準動画ファイル名は、次の形式とする。



```text

yyyymmdd\_role\_session\_take\_camera.mp4

```



代表例:



```text

20260719\_no1\_s01\_t01\_cam01.mp4

```



各要素:



```text

yyyymmdd = 撮影日

role     = 対象役割

sNN      = セッション番号

tNN      = テイク番号

camNN    = カメラ番号

```



\---



\## 28. Version 0.1の制限事項



Version 0.1の命名規則には、次の制限がある。



\- 撮影場所をファイル名に含めない

\- 参加者IDをファイル名に含めない

\- 天候をファイル名に含めない

\- カメラ角度をファイル名に含めない

\- 解像度やFPSをファイル名に含めない

\- 成功・失敗状態をファイル名に含めない



これらの情報は、メタデータJSONへ記録する。



ファイル名へ情報を詰め込みすぎると、入力ミスや管理負担が増えるため、Version 0.1では必要最小限とする。
